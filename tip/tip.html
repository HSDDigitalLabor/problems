Tip Calculator

Durch die zunehmende Nutzung bargeldloser Zahlungen gibt es immer mehr digitale Terminals, die die Möglichkeit bieten, ein Trinkgeld direkt am Gerät auszuwählen.

Erstellen Sie eine Datei namens tip.py und schreiben Sie ein Python-Programm, das für einen eingegebenen Rechnungsbetrag die Trinkgeldbeträge für 5 %, 10 % und 15 % berechnet und ausgibt. Das Programm soll dabei das eingegebene Währungssymbol (z. B. $ oder €) erkennen und dieses auch bei der Ausgabe der Trinkgeldbeträge verwenden.

Hinweise
  • Denken Sie daran, dass input einen str zurückgibt, siehe docs.python.org/3/library/functions.html#input.
  • Denken Sie daran, dass float einen str in einen float konvertieren kann, siehe docs.python.org/3/library/functions.html#float.
  • Denken Sie daran, dass ein str viele Methoden besitzt, siehe docs.python.org/3/library/stdtypes.html#string-methods .

Bevor Sie beginnen

Melden Sie sich bei cs50.dev an, klicken Sie auf Ihr Terminal und führen Sie cd ohne Parameter aus. Sie sollten feststellen, dass der Prompt Ihres Terminals wie unten aussieht:

$

Als nächstes führen Sie

mkdir tip

aus, um einen Ordner namens tip in Ihrem Codespace zu erstellen.

Führen Sie anschließend

cd tip

aus, um in dieses Verzeichnis zu wechseln. Der Prompt ihres Terminals sollte nun tip/$ anzeigen. Jetzt können Sie

code tip.py

ausführen, um eine Datei namens tip.py zu erstellen, in der Sie Ihr Programm schreiben. Der Anfang von dem Programm wurde bereits erstellt, kopieren Sie diesen Code in Ihre Datei tip.py und vervollständigen Sie den Code.

So testen Sie Ihr Programm

So können Sie Ihr Programm manuell testen:

  • Führen Sie Ihr Programm mit python tip.py aus. Ihr Programm sollte ausgeben: Rechnungsbetrag eingeben: und auf die Eingabe des Benutzers warten. Geben Sie 28.70€ ein und drücken Sie Enter. Ihr Programm sollte ausgeben:
    Tip 5%: 1.44€
    Tip 10%: 2.87€
    Tip 15%: 4.30€
  • Führen Sie Ihr Programm mit python tip.py aus. Ihr Programm sollte ausgeben: Rechnungsbetrag eingeben: und auf die Eingabe des Benutzers warten. Geben Sie 145.75$ ein und drücken Sie Enter. Ihr Programm sollte ausgeben:
    Tip 5%: 7.29$
    Tip 10%: 14.57$
    Tip 15%: 21.86$

Sie können das folgende Kommando ausführen, um Ihren Code mit check50 zu überprüfen, einem Programm, das bei der Abgabe verwendet wird um Ihren Code zu testen. Testen Sie Ihr Programm aber auch selbst!

check50 HSDDigitalLabor/problems/git2025/tip

Grüne Smileys :) bedeuten, dass Ihr Programm einen Test bestanden hat! Rote traurige Smileys :( zeigen an, dass Ihr Programm etwas Unerwartetes ausgegeben hat. Besuchen Sie die URL, die check50 ausgibt, um zu sehen, welche Eingabe check50 an Ihr Programm übergeben hat, welche Ausgabe erwartet wurde und welche Ausgabe Ihr Programm tatsächlich geliefert hat.

Abgabe

Führen Sie im Terminal den folgenden Befehl aus, um Ihre Arbeit einzureichen.

submit50 HSDDigitalLabor/problems/git2025/tip

Führen Sie diesen Befehl vor dem Fälligkeitsdatum aus. Sie können Ihre Lösung nach dem ersten Einsenden noch ändern und erneut einreichen. Bewertet wird die zuletzt eingereichte Version vor dem Fälligkeitsdatum. Nach Ablauf der Frist können Sie technisch zwar noch Abgaben tätigen, Ihre Lösung wird jedoch nicht mehr gewertet. Das Ergebnis kann ausschließlich mit obigem Befehl abgegeben werden, eine Abgabe in Moodle ist nicht möglich.
