Zahlenjagd

Aufgabe

Erstellen Sie eine Datei mit dem Namen zahlenjagd.py und schreiben Sie ein Programm, das eine Zahlenjagd für einen Spieler implementiert. Das Programm wählt geheim eine zufällige Zahl zwischen 1 und 100 aus, die der Spieler durch wiederholtes Raten ermitteln muss. Nach jedem Versuch gibt das Programm Rückmeldung, ob die geratene Zahl zu hoch, zu niedrig oder korrekt ist. Am Ende gibt das Programm die Anzahl der Versuche aus.

Hinweise
  • Zufällige Zahl generieren: Zu Beginn wählt das Programm zufällig eine Zahl zwischen 1 und 100 aus.
  • Eingabe des Spielers: Der Spieler wird wiederholt aufgefordert, eine Zahl einzugeben, um die geheime Zahl zu erraten.
  • Hinweise geben: Nach jedem Versuch informiert das Programm, ob die Eingabe zu hoch, zu niedrig oder korrekt ist.
    • Versuch > gesuchte Zahl terminal Ausgabe: "zuhoch"
    • Versuch < gesuchte Zahl terminal Ausgabe: "zuniedrig"
    • Korrekt: Ausgabe einer Erfolgsmeldung und Beenden des Spiels.
  • Versuche zählen: Das Programm zählt die Anzahl der Versuche und gibt diese nach dem Erraten aus.
  • Fehlerbehandlung: Eingaben werden geprüft — nur ganze Zahlen zwischen 1 und 100 sind gültig. Ungültige Eingaben lösen eine Fehlermeldung aus und führen zu einem neuen Eingabeversuch.

Bevor Sie beginnen

Melden Sie sich bei cs50.dev an, klicken Sie auf Ihr Terminal und führen Sie cd ohne Parameter aus. Sie sollten feststellen, dass der Prompt Ihres Terminals wie unten aussieht:

$

Als nächstes führen Sie

mkdir zahlenjagd

aus, um einen Ordner namens zahlenjagd in Ihrem Codespace zu erstellen.

Führen Sie anschließend

cd zahlenjagd

aus, um in dieses Verzeichnis zu wechseln. Der Prompt ihres Terminals sollte nun zahlenjagd/$ anzeigen. Jetzt können Sie

code zahlenjagd.py

ausführen, um eine Datei namens zahlenjagd.py zu erstellen, in der Sie Ihr Programm schreiben.

Abgabe

Führen Sie im Terminal den folgenden Befehl aus, um Ihre Arbeit einzureichen.

Führen Sie diesen Befehl vor dem Fälligkeitsdatum aus. Sie können Ihre Lösung nach dem ersten Einsenden noch ändern und erneut einreichen. Bewertet wird die zuletzt eingereichte Version vor dem Fälligkeitsdatum. Nach Ablauf der Frist können Sie technisch zwar noch Abgaben tätigen, Ihre Lösung wird jedoch nicht mehr gewertet. Das Ergebnis kann ausschließlich mit obigem Befehl abgegeben werden, eine Abgabe in Moodle ist nicht möglich.
