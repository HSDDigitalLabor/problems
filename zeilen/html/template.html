Bei der Lösung von linearen Gleichungssystemen mit dem Gauß-Algorithmus werden die Gleichungen in eine Matrixform gebracht. Zur weiteren Vereinfachung werden dabei sogenannte Zeilenoperationen angewendet. Die elementaren Zeilenoperationen sind:

  • Vertauschen zweier Zeilen
  • Multiplikation einer Zeile mit einer von Null verschiedenen Zahl
  • Addition eines Vielfachen einer Zeile zu einer anderen Zeile

Für die nachfolgenden Probleme zum Gauß-Algorithmus werden die letzten beiden Zeilenoperationen benötigt.

Aufgabe

Implementieren Sie diese zwei elementaren Zeilenoperationen als Funktionen in Python. Die Funktionen bekommen die Matrix M als Liste von Listen übergeben und sollen die Matrix M in-place verändern.

Die äußere Liste stellt die Zeilen dar, die inneren Listen repräsentieren die Spalten. Diese beiden Funktionen sollen implementiert werden:

  • rowMult(M, i, k)
  • rowAdd(M, i, j, k)

{% include "file.html" %} implementieren Sie die folgenden Funktionen in Ihrer Datei: 

def rowMult(M, i, k):
    """Multipliziert die i-te Zeile der Matrix M mit dem Faktor k."""


def rowAdd(M, i, j, k):
    """Addiert das k-fache der i-ten Zeile zur j-ten Zeile der Matrix M."""

Beachten Sie dabei folgende Anforderungen:

  • Die Matrix M wird als Liste von Listen übergeben (äußere Liste = Zeilen, innere Listen = Spalten).
  • Die Operationen sollen die Matrix in-place verändern (keine neue Matrix zurückgeben).
  • i und j sind Zeilenindizes (0-basiert).
  • Überprüfen Sie die Indizes i und j auf Gültigkeit, um Fehler zu vermeiden.
  • k ist ein Faktor (Zahl), mit dem multipliziert bzw. skaliert wird.
  • Falls k den Wert 0 hat, soll die Matrix nicht verändert werden.
  • Hinweis: Dies ist eine Vereinfachung für diese Aufgabe. In der Mathematik würde das Multiplizieren einer Zeile mit 0 normalerweise dazu führen, dass die gesamte Zeile zu Nullen wird. Für diese Übung soll die Matrix jedoch bei k = 0 unverändert bleiben, wie in den Tests vorgegeben.
Hinweise
  • In Python werden List-Objekte als Referenzen übergeben, daher wirken sich Änderungen in den Funktionen direkt auf die ursprüngliche Matrix aus.
