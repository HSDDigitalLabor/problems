Wiedergabegeschwindigkeit

Manche Menschen neigen dazu, beim Vortragen sehr schnell zu sprechen, und es wäre schön, sie zu verlangsamen – ähnlich wie bei YouTubes 0,75-facher Wiedergabegeschwindigkeit oder indem sie zwischen den Wörtern pausieren.

Erstellen Sie eine Datei mit dem Namen playback.py und implementieren Sie ein Python-Programm, das den Benutzer nach einer Eingabe fragt und anschließend genau diese Eingabe ausgibt, wobei jedes Leerzeichen durch ... (drei Punkte) ersetzt wird.

Hinweise
  • Denken Sie daran, dass Sie mit input einen str erhalten, siehe docs.python.org/3/library/functions.html#input.
  • Denken Sie daran, dass Sie einen str mit print ausgeben können, siehe
  • Beachten Sie, dass ein str über zahlreiche Methoden verfügt, siehe . docs.python.org/3/library/stdtypes.html#string-methods

Bevor Sie beginnen

Melden Sie sich bei cs50.dev an, klicken Sie auf Ihr Terminal und führen Sie cd ohne Parameter aus. Sie sollten feststellen, dass der Prompt Ihres Terminals wie unten aussieht:

$

Als nächstes führen Sie

mkdir playback

aus, um einen Ordner namens playback in Ihrem Codespace zu erstellen.

Führen Sie anschließend

cd playback

aus, um in dieses Verzeichnis zu wechseln. Der Prompt ihres Terminals sollte nun playback/$ anzeigen. Jetzt können Sie

code playback.py

ausführen, um eine Datei namens playback.py zu erstellen, in der Sie Ihr Programm schreiben.

So testen Sie Ihr Programm

So können Sie Ihr Programm manuell testen:

  • Führen Sie Ihr Programm mit python playback.py aus. Ihr Programm sollte ausgeben: Ihre Eingabe: und auf die Eingabe des Benutzers warten. Geben Sie Grundlagen der Informatik II ein und drücken Sie Enter. Ihr Programm sollte ausgeben:
    Grundlagen...der...Informatik...II
  • Führen Sie Ihr Programm mit python playback.py aus. Ihr Programm sollte ausgeben: Ihre Eingabe: und auf die Eingabe des Benutzers warten. Geben Sie Python ist einfach zu lernen und vielseitig einsetzbar. ein und drücken Sie Enter. Ihr Programm sollte ausgeben:
    Python...ist...einfach...zu...lernen...und...vielseitig...einsetzbar.

Sie können das folgende Kommando ausführen, um Ihren Code mit check50 zu überprüfen, einem Programm, das bei der Abgabe verwendet wird um Ihren Code zu testen. Testen Sie Ihr Programm aber auch selbst!

check50 HSDDigitalLabor/problems/git2025/playback

Grüne Smileys :) bedeuten, dass Ihr Programm einen Test bestanden hat! Rote traurige Smileys :( zeigen an, dass Ihr Programm etwas Unerwartetes ausgegeben hat. Besuchen Sie die URL, die check50 ausgibt, um zu sehen, welche Eingabe check50 an Ihr Programm übergeben hat, welche Ausgabe erwartet wurde und welche Ausgabe Ihr Programm tatsächlich geliefert hat.

Abgabe

Führen Sie im Terminal den folgenden Befehl aus, um Ihre Arbeit einzureichen.

submit50 HSDDigitalLabor/problems/git2025/playback

Führen Sie diesen Befehl vor dem Fälligkeitsdatum aus. Sie können Ihre Lösung nach dem ersten Einsenden noch ändern und erneut einreichen. Bewertet wird die zuletzt eingereichte Version vor dem Fälligkeitsdatum. Nach Ablauf der Frist können Sie technisch zwar noch Abgaben tätigen, Ihre Lösung wird jedoch nicht mehr gewertet. Das Ergebnis kann ausschließlich mit obigem Befehl abgegeben werden, eine Abgabe in Moodle ist nicht möglich.
