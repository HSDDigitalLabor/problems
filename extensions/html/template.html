Auch wenn Windows und MacOS sie manchmal ausblenden, besitzen die meisten Dateien sogenannte Dateierweiterungen , ein Suffix, das mit einem Punkt (.) am Ende ihres Namens beginnt. Beispielsweise enden Dateinamen für GIFs mit .gif, und Dateinamen für JPEGs mit .jpg oder .jpeg. Wenn Sie auf eine Datei doppelklicken, verwendet Ihr Computer die Dateierweiterung, um zu bestimmen, welches Programm zum Öffnen der Datei gestartet werden soll.

Webbrowser hingegen verwenden Medientypen, früher als MIME-Typen bekannt, um zu bestimmen, wie Dateien im Web angezeigt werden. Wenn Sie eine Datei von einem Webserver herunterladen, sendet dieser zusammen mit der Datei einen HTTP-Header , der den Medientyp der Datei angibt. Beispielsweise ist der Medientyp für ein GIF image/gif und für ein JPEG image/jpeg. Um den Medientyp einer Datei zu bestimmen, betrachtet ein Webserver in der Regel die Dateierweiterung und ordnet diese einem entsprechenden Medientyp zu.

Siehe developer.mozilla.org/en-US/docs/Web/HTTP/Basics_of_HTTP/MIME_types/Common_types für gängige Typen.

{% include "file.html" %} implementieren Sie ein Programm, das den Benutzer nach dem Namen einer Datei fragt und anschließend den Medientyp der Datei ausgibt, sofern der Dateiname (unabhängig von Groß- und Kleinschreibung) mit einem der folgenden Suffixe endet:

  • .gif
  • .jpg
  • .jpeg
  • .png
  • .pdf
  • .txt
  • .zip

Wenn der Dateiname mit einem anderen Suffix endet oder gar kein Suffix besitzt, geben Sie stattdessen application/octet-stream aus, was ein häufiger Standardwert ist.

Hinweise
  • Denken Sie daran, dass ein str mit einigen Methoden ausgestattet ist, siehe docs.python.org/3/library/stdtypes.html#string-methods .
  • Dateinamen dürfen mehrere Punkte enthalten. Für die Bestimmung der Dateierweiterung ist immer der letzte Punkt entscheidend: Alles hinter diesem letzten Punkt ist die Dateierweiterung. Siehe Wikipedia – Dateiname.
