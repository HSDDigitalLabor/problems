Dateierweiterungen

Auch wenn Windows und MacOS sie manchmal ausblenden, besitzen die meisten Dateien sogenannte Dateierweiterungen , ein Suffix, das mit einem Punkt (.) am Ende ihres Namens beginnt. Beispielsweise enden Dateinamen für GIFs mit .gif, und Dateinamen für JPEGs mit .jpg oder .jpeg. Wenn Sie auf eine Datei doppelklicken, verwendet Ihr Computer die Dateierweiterung, um zu bestimmen, welches Programm zum Öffnen der Datei gestartet werden soll.

Webbrowser hingegen verwenden Medientypen, früher als MIME-Typen bekannt, um zu bestimmen, wie Dateien im Web angezeigt werden. Wenn Sie eine Datei von einem Webserver herunterladen, sendet dieser zusammen mit der Datei einen HTTP-Header , der den Medientyp der Datei angibt. Beispielsweise ist der Medientyp für ein GIF image/gif und für ein JPEG image/jpeg. Um den Medientyp einer Datei zu bestimmen, betrachtet ein Webserver in der Regel die Dateierweiterung und ordnet diese einem entsprechenden Medientyp zu.

Siehe developer.mozilla.org/en-US/docs/Web/HTTP/Basics_of_HTTP/MIME_types/Common_types für gängige Typen.

Erstellen Sie eine Datei mit dem Namen extensions.py und implementieren Sie ein Programm, das den Benutzer nach dem Namen einer Datei fragt und anschließend den Medientyp der Datei ausgibt, sofern der Dateiname (unabhängig von Groß- und Kleinschreibung) mit einem der folgenden Suffixe endet:

  • .gif
  • .jpg
  • .jpeg
  • .png
  • .pdf
  • .txt
  • .zip

Wenn der Dateiname mit einem anderen Suffix endet oder gar kein Suffix besitzt, geben Sie stattdessen application/octet-stream aus, was ein häufiger Standardwert ist.

Hinweise
  • Denken Sie daran, dass ein str mit einigen Methoden ausgestattet ist, siehe docs.python.org/3/library/stdtypes.html#string-methods .

Demo

Bevor Sie beginnen

Melden Sie sich bei cs50.dev an, klicken Sie auf Ihr Terminal und führen Sie cd ohne Parameter aus. Sie sollten feststellen, dass der Prompt Ihres Terminals wie unten aussieht:

$

Als nächstes führen Sie

mkdir extensions

aus, um einen Ordner namens extensions in Ihrem Codespace zu erstellen.

Führen Sie anschließend

cd extensions

aus, um in dieses Verzeichnis zu wechseln. Der Prompt ihres Terminals sollte nun extensions/$ anzeigen. Jetzt können Sie

code extensions.py

ausführen, um eine Datei namens extensions.py zu erstellen, in der Sie Ihr Programm schreiben.

So testen Sie Ihr Programm

So können Sie Ihr Programm manuell testen:

  • Führen Sie Ihr Programm mit python extensions.py aus. Geben Sie happy.jpg ein und drücken Sie Enter. Ihr Programm sollte ausgeben:
    image/jpeg
  • Führen Sie Ihr Programm mit python extensions.py aus. Geben Sie document.pdf ein und drücken Sie Enter. Ihr Programm sollte ausgeben:
    application/pdf

Testen Sie unbedingt alle anderen Dateiformate, variieren Sie die Groß- und Kleinschreibung Ihrer Eingabe und fügen Sie „versehentlich“ Leerzeichen auf beiden Seiten Ihrer Eingabe hinzu, bevor Sie Enter drücken. Ihr Programm sollte wie erwartet funktionieren, unabhängig von Groß-/Kleinschreibung und Leerzeichen.

Sie können das folgende Kommando ausführen, um Ihren Code mit check50 zu überprüfen, einem Programm, das bei der Abgabe verwendet wird um Ihren Code zu testen. Testen Sie Ihr Programm aber auch selbst!

check50 HSDDigitalLabor/problems/git2025/extensions

Grüne Smileys bedeuten, dass Ihr Programm einen Test bestanden hat! Rote traurige Smileys zeigen an, dass Ihr Programm etwas Unerwartetes ausgegeben hat. Besuchen Sie die URL, die check50 ausgibt, um zu sehen, welche Eingabe check50 an Ihr Programm übergeben hat, welche Ausgabe erwartet wurde und welche Ausgabe Ihr Programm tatsächlich geliefert hat.

Abgabe

Führen Sie im Terminal den folgenden Befehl aus, um Ihre Arbeit einzureichen.

submit50 HSDDigitalLabor/problems/git2025/extensions

Führen Sie diesen Befehl vor dem Fälligkeitsdatum aus. Sie können Ihre Lösung nach dem ersten Einsenden noch ändern und erneut einreichen. Bewertet wird die zuletzt eingereichte Version vor dem Fälligkeitsdatum. Nach Ablauf der Frist können Sie technisch zwar noch Abgaben tätigen, Ihre Lösung wird jedoch nicht mehr gewertet. Das Ergebnis kann ausschließlich mit obigem Befehl abgegeben werden, eine Abgabe in Moodle ist nicht möglich.
