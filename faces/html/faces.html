Emoticons zu Emojis umwandeln

Bevor es Emojis gab, wurden sogenannte Emoticons genutzt, also Zeichenfolgen wie :) für ein lachendes und :( für ein trauriges Gesicht. Heute werden solche Emoticons von Programmen meist automatisch in Emojis umgewandelt.

Erstellen Sie eine Datei mit dem Namen faces.py und schreiben Sie ein Python-Programm, das mithilfe der Funktion convert die beschriebene Umwandlung implementiert. Diese Funktion erhält einen str als Eingabe und gibt einen konvertierten str zurück. Bei der Konvertierung wird jede Zeichenfolge ":)" durch 🙂 und jede Zeichenfolge ":(" durch 🙁 ersetzt, alle anderen Zeichen bleiben unverändert.

Hinweise
  • Denken Sie daran, dass input einen str zurückgibt docs.python.org/3/library/functions.html#input.
  • Denken Sie daran, dass ein str mit einigen Methoden ausgestattet ist, siehe docs.python.org/3/library/stdtypes.html#string-methods .
  • Ein Emoji ist tatsächlich nur ein Zeichen, Sie können es also wie jeden str zitieren, z. B. "😐". Sie können das Emoji von dieser Seite kopieren und in Ihren eigenen Code einfügen.
  • Emojis sind Teil des Unicode-Standards. Weitere Informationen finden Sie unter Unicode for Smileys . In Python können Unicode-Zeichen mit folgender Notation verarbeitet werden: "\U0001F600" oder "\u0394". Mehr dazu unter docs.python.org/3/howto/unicode.html .

Bevor Sie beginnen

Melden Sie sich bei cs50.dev an, klicken Sie auf Ihr Terminal und führen Sie cd ohne Parameter aus. Sie sollten feststellen, dass der Prompt Ihres Terminals wie unten aussieht:

$

Als nächstes führen Sie

mkdir faces

aus, um einen Ordner namens faces in Ihrem Codespace zu erstellen.

Führen Sie anschließend

cd faces

aus, um in dieses Verzeichnis zu wechseln. Der Prompt ihres Terminals sollte nun faces/$ anzeigen. Jetzt können Sie

code faces.py

ausführen, um eine Datei namens faces.py zu erstellen, in der Sie Ihr Programm schreiben.

So testen Sie Ihr Programm

So können Sie Ihr Programm manuell testen:

  • Führen Sie Ihr Programm mit python faces.py aus. Ihr Programm sollte ausgeben: Geben Sie einen Text mit Emoticons ein: und auf die Eingabe des Benutzers warten. Geben Sie Hallo :) ein und drücken Sie Enter. Ihr Programm sollte ausgeben:
    Hallo 🙂
  • Führen Sie Ihr Programm mit python faces.py aus. Ihr Programm sollte ausgeben: Geben Sie einen Text mit Emoticons ein: und auf die Eingabe des Benutzers warten. Geben Sie Tschüß :( ein und drücken Sie Enter. Ihr Programm sollte ausgeben:
    Tschüß 🙁

Sie können das folgende Kommando ausführen, um Ihren Code mit check50 zu überprüfen, einem Programm, das bei der Abgabe verwendet wird um Ihren Code zu testen. Testen Sie Ihr Programm aber auch selbst!

check50 HSDDigitalLabor/problems/git2025/faces

Grüne Smileys :) bedeuten, dass Ihr Programm einen Test bestanden hat! Rote traurige Smileys :( zeigen an, dass Ihr Programm etwas Unerwartetes ausgegeben hat. Besuchen Sie die URL, die check50 ausgibt, um zu sehen, welche Eingabe check50 an Ihr Programm übergeben hat, welche Ausgabe erwartet wurde und welche Ausgabe Ihr Programm tatsächlich geliefert hat.

Abgabe

Führen Sie im Terminal den folgenden Befehl aus, um Ihre Arbeit einzureichen.

submit50 HSDDigitalLabor/problems/git2025/faces

Führen Sie diesen Befehl vor dem Fälligkeitsdatum aus. Sie können Ihre Lösung nach dem ersten Einsenden noch ändern und erneut einreichen. Bewertet wird die zuletzt eingereichte Version vor dem Fälligkeitsdatum. Nach Ablauf der Frist können Sie technisch zwar noch Abgaben tätigen, Ihre Lösung wird jedoch nicht mehr gewertet. Das Ergebnis kann ausschließlich mit obigem Befehl abgegeben werden, eine Abgabe in Moodle ist nicht möglich.
