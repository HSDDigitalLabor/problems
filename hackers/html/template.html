Bei der Bearbeitung des Problem "iban.py" haben Sie bereits die Struktur und die Prüfziffernberechnung einer IBAN kennengelernt, sollten Sie dieses Problem noch nicht bearbeitet haben, bearbeiten Sie dieses zuerst. In dieser Aufgabe sollen Sie eine Liste von IBANs verarbeiten, die aus einem kompromittierten "Hackers News Forum" stammt. Ein anonymer Whistleblower hat Ihnen eine Datei mit verdächtigen IBAN-Nummern zugespielt, die möglicherweise für betrügerische Zwecke missbraucht werden sollten. Ihre Aufgabe ist es, diese Liste zu analysieren und die Anzahl der ungültigen IBANs zu ermitteln, um das Ausmaß des Datenlecks besser einschätzen zu können.

Aufgabe

{% include "file.html" %} implementieren Sie eine Funktion def checkHackersList(listenname):, die als Argument einen Dateinamen (type String) übergeben bekommt und diese aus dem aktuellen Verzeichnis einliest. Die Datei enthält eine Liste von IBANs, jeweils eine pro Zeile. Öffnen Sie diese Datei lesend und überprüfen Sie jede IBAN auf Gültigkeit. Verwenden Sie dazu die in der letzten Aufgabe implementierte Funktion def calcCheckDigitsDE(blz, konto): zur IBAN-Prüfziffernberechnung. Die Funktion soll die Anzahl der ungültigen IBANs in der Datei zählen und als Ganzzahl int zurückgeben.
(Die Gültigkeit einer IBAN bezieht sich ausschließlich auf die korrekte Berechnung der Prüfziffer, nicht auf die tatsächliche Existenz des Kontos. Alle in diesem Problem verwendeten IBANs sind fiktiv.)

Hinweise
  • Denken Sie daran, in Python können Dateien einfach gelesen werden, siehe https://docs.python.org/3/tutorial/inputoutput.html#reading-and-writing-files .
  • Denken Sie daran, Sie können eine Schleife verwenden, um auf einzelne Zeilen einer Datei zuzugreifen. Siehe https://docs.python.org/3/tutorial/controlflow.html#for-statements .
  • Denken Sie daran, das es möglich ist auf einzelne Teile eines Strings zuzugreifen, siehe docs.python.org/3/tutorial/introduction.html#text .
