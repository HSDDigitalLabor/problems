Hallo

Erstellen Sie eine Datei mit dem Namen hello.py und implementieren Sie ein Python-Programm, das den Benutzer nach seinem Namen fragt und anschließend Hallo, so-und-so ausgibt, wobei so-und-so durch den eingegebenen Namen ersetzt wird.

Hinweise
  • Denken Sie daran, dass input einen str zurückgibt, siehe docs.python.org/3/library/functions.html#input.
  • Denken Sie daran, dass Sie einen str mit print ausgeben können.
  • Denken Sie daran, dass Sie in Python formatierte Strings erstellen können, indem Sie dem String ein f voranstellen. Zum Beispiel ersetzt f"{name}" den Wert der Variablen name an der Stelle, an der Sie {name} geschrieben haben.

Bevor Sie beginnen

Melden Sie sich bei cs50.dev an, klicken Sie auf Ihr Terminal und führen Sie cd ohne Parameter aus. Sie sollten feststellen, dass der Prompt Ihres Terminals wie unten aussieht:

$

Als nächstes führen Sie

mkdir hello

aus, um einen Ordner namens hello in Ihrem Codespace zu erstellen.

Führen Sie anschließend

cd hello

aus, um in dieses Verzeichnis zu wechseln. Der Prompt ihres Terminals sollte nun hello/$ anzeigen. Jetzt können Sie

code hello.py

ausführen, um eine Datei namens hello.py zu erstellen, in der Sie Ihr Programm schreiben.

So testen Sie Ihr Programm

So können Sie Ihr Programm manuell testen:

  • Führen Sie Ihr Programm mit python hello.py aus. Ihr Programm sollte ausgeben: Wie heißt du? und auf die Eingabe des Benutzers warten. Geben Sie Markus ein und drücken Sie Enter. Ihr Programm sollte ausgeben:
    Hallo, Markus
  • Führen Sie Ihr Programm mit python hello.py aus. Ihr Programm sollte ausgeben: Wie heißt du? und auf die Eingabe des Benutzers warten. Geben Sie Sophia ein und drücken Sie Enter. Ihr Programm sollte ausgeben:
    Hallo, Sophia
  • Führen Sie Ihr Programm mit python hello.py aus. Ihr Programm sollte ausgeben: Wie heißt du? und auf die Eingabe des Benutzers warten. Geben Sie Noah ein und drücken Sie Enter. Ihr Programm sollte ausgeben:
    Hallo, Noah

Abgabe

Führen Sie im Terminal den folgenden Befehl aus, um Ihre Arbeit einzureichen.

submit50 HSDDigitalLabor/problems/git2025/hello

Führen Sie diesen Befehl vor dem Fälligkeitsdatum aus. Sie können Ihre Lösung nach dem ersten Einsenden noch ändern und erneut einreichen. Bewertet wird die zuletzt eingereichte Version vor dem Fälligkeitsdatum. Nach Ablauf der Frist können Sie technisch zwar noch Abgaben tätigen, Ihre Lösung wird jedoch nicht mehr gewertet. Das Ergebnis kann ausschließlich mit obigem Befehl abgegeben werden, eine Abgabe in Moodle ist nicht möglich.
